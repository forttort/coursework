# RINKAN FIRST SOURCE

## Решение
Для первой версии проекта первым источником для парсинга выбран **RINKAN**.

## Почему это решение уже подтверждено репозиторием
Под `RINKAN` уже существуют:
- парсер `parser/rinkan_parser_v4.py`
- sample-данные в `data/products.sample.json`
- схема БД `sql/schema_v2.sql`
- frontend/backend MVP для просмотра товаров

## Что это дало проекту
- вместо абстрактной модели появился конкретный набор полей
- стало понятно, как строить категории и подкатегории
- появилась база для MVP каталога и страницы товара

## Что дальше
Дальнейшая разработка должна продолжаться вокруг `RINKAN` до завершения первого полного контура:

1. parser
2. loader
3. PostgreSQL
4. backend от БД
5. frontend поверх реального API
