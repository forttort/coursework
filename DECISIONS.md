# DECISIONS

## Принятые решения

### 1. Тематика проекта
Проект ориентирован на товары fashion-сегмента:
- одежда
- обувь
- сумки
- аксессуары
- украшения
- часы

### 2. Формат проекта
Проект реализуется как веб-каталог товаров.

### 3. Формат интерфейса
Система должна включать:
- каталог товаров
- карточки товаров
- поиск
- фильтрацию по категориям
- страницу товара с подробной информацией

### 4. Источники данных
В качестве основных источников данных рассматриваются:
- Mercari Japan
- Yahoo! Auctions Japan
- Rakuten Rakuma
- 2nd STREET
- RINKAN

### 5. Приоритет разработки
Порядок работы:
1. Зафиксировать концепцию проекта
2. Определить модель данных товара
3. Спроектировать базу данных
4. Реализовать парсер для одного источника
5. Реализовать backend API
6. Реализовать frontend

### 6. Текущее состояние
На данный момент:
- определена общая идея проекта
- определена тематика проекта
- определены категории товаров
- создана базовая структура папок

Следующий этап:
- определить структуру сущности товара
- спроектировать таблицы базы данных
- выбрать первый источник для парсинга
