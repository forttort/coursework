# PROJECT CONTEXT

## Рабочая идея проекта
Создать веб-каталог отобранных товаров с японских сайтов.

На первом этапе проект рассматривается как система, которая:
- получает данные о товарах из внешних источников
- фильтрует и отбирает подходящие позиции
- хранит данные в базе данных
- отображает товары в виде каталога в веб-интерфейсе

В перспективе каталог должен использоваться как витрина товаров, через которую пользователь сможет выбрать интересующую позицию и оформить заказ через владельца сервиса.

## Предварительная архитектура
Python parser → PostgreSQL → FastAPI → web-client

## Что уже решено
- проект будет связан с товарами с японских сайтов
- клиентская часть будет веб-интерфейсом
- в качестве базы данных планируется PostgreSQL
- серверная часть будет реализована на FastAPI
- товары перед публикацией должны проходить фильтрацию
- каталог ориентирован на отобранные позиции, а не на весь поток данных
- в дальнейшем система может использоваться для оформления заказов через посредника

## Что пока не определено точно
- окончательный список функций
- точный состав таблиц базы данных
- какие сайты будут подключены первыми
- какие поля товара обязательны
- по каким правилам будет происходить фильтрация товаров
- как именно будет реализовано оформление заказа

## Текущий подход
Сейчас репозиторий создается как черновая база проекта, чтобы постепенно уточнять структуру, цели и реализацию.
